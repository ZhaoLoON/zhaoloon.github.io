Java NIO

发表于 2015-05-10

继续复习一下Java基础知识，这个文章算是阅读笔记吧。

概述

Java1.4引入Java NIO(Non-blocking I/O 或者New I/O)，为Java提供了非阻塞IO，并且在Java 1.7中又引入了JAVA NIO.2，位Java增加了异步通道和新的文件心痛API。

  • Channels and Buffers

标准的IO是基于字节流和字符流进行操作，二NIO是基于Channel和Buffer进行操作，数据总是从通道读取到缓冲区中，或者从缓冲区写入到通道中。

  • Non-blocking IO

Java NIO让你以非阻塞的方式使用IO，例如：当线程从通道读取数据到缓冲区时，线程还是可以进行其他属性。当数据被写入到缓冲区时，线程可以继续处理它。从缓冲区写入通道也类似。

  • Selector

Java NIO引入了选择器的概念，选择器用于监听多个通道的事件(比如连接打开)。因此单个线程可以监听多个数据通道。需要使用Selector，需要向Selector注册Channel，然后调用它的select()方法。这个方法会一直阻塞到某个注册的通道有事件就绪。一旦这个方法返回，线程就可以处理这些事件。

  • Java NIO一些主要的Channel

    • FileChannel 从文件中读写数据
    • DatagramChannel 通过UDP读写网络数据
    • SocketChannel 通过TCP读写网络数据
    • ServerSocketChannel 可以监听新进来的TCP连接，对每个新进来的连接都会创建一个SocketChannel

  • Java NIO中关键的Buffer

    • ByteBuffer
    • CharBuffer
    • DoubleBuffer
    • FloatBuffer
    • IntBuffer
    • LongBuffer
    • ShortBuffer
    • MappedByteBuffer

Channel

Java NIO的Channel类似流，但又有些不同：

  • 既可以从通道中读取数据，又可以写数据到通道中。但流的读写通常是单向的。
  • 通道可以异步读写。
  • 通道中的数据总是要先读取一个Buffer，或者总是要从一个Buffer中写入。

Buffer

Buffer本质上是一块可以写入数据，然后可以从中读取数据的内存。这块内存被包装成NIO Buffer对象，并提供了一组方法，用于方便的访问块内存。

Buffer的基本用法

Buffer读写数据一下四个步骤：

  1. 写入数据到Buffer
  2. 调用flip()方法
  3. 从Buffer中读取数据
  4. 调用clear()方法或者compact()方法

向Buffer中写入数据时，buffer会记录下写了多少数据。一旦要读取数据，需要通过flip()方法将Buffer从写模式切换到读模式。在读模式下，可以读取之前写入的数据。
一旦读完所有数据，就需要清空缓存区，让它可以被再次写入。有两种方法清空缓冲区：

  • clear()方法会清空整个缓冲区。
  • compact()方法只会清空以读过的数据，任何未读的数据都被移到缓冲区的起始处，新写入的数据将放到缓冲区未读数据的后面。

Buffer的capacity, position和limit

理解Buffer的工作原理，需要熟悉它的三个属性：
下图转自并发编程网

  • capacity

    作为一个内存块，Buffer有一个固定的大小值，就是capacity。只能往里写capacity个byte、long、char等类型。一旦buffer满了，需要将其清空才能继续写数据。

  • position

    • 写模式：初始值为0，当没写入一个数据时，position向前移动到下一个可以插入的数据的Buffer单元。position最大值为capacity-1。
    • 读模式： 当Buffer从写模式切换到读模式，position会被重置为0，当从Buffer的position处读取数据时，position向前移动到下一个可以读的位置。

  • limit

    • 写模式：limit表示最多能往buffer里写多少数据，此时limit=capacity
    • 读模式：切换到读模式时，limit表示最多可以读到多少数据。此时limit会被设置成写模式下的position值。

Buffer的类型

Java NIO有以下Buffer类型

  • ByteBuffer
  • MappedByteBuffer
  • CharBuffer
  • DoubleBuffer
  • FloatBuffer
  • IntBuffer
  • LongBuffer
  • ShortBuffer

Buffer的分配

想要获得一个Buffer对象首先要进行分配。每个buffer有一个allocate方法。

向Buffer中写入数据

两种方式向Buffer中写入数据：

  • 从Channel写到Buffer
  • 通过Buffer的put()方法写到Buffer里。

flip()方法

flip方法将Buffer从写模式切换到读模式。调用flip方法会将position设回0，并将limit设置成之前position的值。
换句话说，position现在用于标记读的位置，limit表示之前写进了多少个byte、char等。

从Buffer中读取数据

两种方法从Buffer中读取数据：

  • 从Buffer读取数据到Channel
  • 使用get()方法从Buffer中读取数据

rewind()方法

rewind()方法将position设回0，可以重读Buffer中的数据。

clear()和compact()

一旦读完Buffer中的数据，需要让Buffer准备好再次被写入。
如果调用clear方法，position将被设回0，limit被设置成capacity的值。换句话说被清空了。Buffer中的数据并未清楚，只是这些标记告诉我们可以从哪里开始往Buffer里写数据。
如果Buffer中仍有未读的数据，且后续还需要这些数据，但是此时想要先写数据，那么使用compact方法。
compact方法将所有未读的数据拷贝到Buffer起始处。然后将position设到最后一个未读元素正后面。limit设置为capacity。

mark()与reset()方法

通过调用Buffer的mark方法，可以标记Buffer中的一个特定position。之后可以通过调用Buffer.reset()方法恢复到这个position。

equals与compareTo方法

equals()

满足一下条件，两个Buffer相等：

  1. 有相同的类型(byte、char、int等)
  2. Buffer中剩余的个数相等
  3. Buffer中所有剩余的byte、char等都相同

compareTo()

如果满足下列条件，则认为一个Buffer“小于”另一个Buffer：

  1. 第一个不相等的元素小于另一个Buffer中对应的元素 。
  2. 所有元素都相等，但第一个Buffer比另一个先耗尽(第一个Buffer的元素个数比另一个少)。

Scatter/Gather

  • Scatter 从Channel中读操作时将读取的数据写入多个buffer中。因此Channel将从Channel中读取的数据分散(scatter)到多个buffer中
  • Gather 写操作是将多个Buffer的数据写入同一个Channel，因此，Channel将多个Buffer中的数据聚集(gather)后发送到Channel

Scatter/Gather经常用于需要将传输的数据分开处理的场合，例如传输一个由消息头和消息体组成的消息，你可能会将消息体和消息头分散到不同的Buffer中，方便的处理消息头和消息体

Scattering Reads

Scattering Reads是指数据从一个channel读取到多个buffer中。
下图转自并发编程网

注意buffer首先被插入到数组，然后再将数组作为channel.read()的输入参数。read()方法按照buffer在数组中的顺序将从channel中读取的数据写入到buffer，当一个buffer被写满后，channel紧接着向另一个buffer中写。

Gathering Writes

Gathering Writes是指数据从多个buffer写入到同一个channel。
下图转自并发编程网

buffers数组是write()方法的入参，write()方法会按照buffer在数组中的顺序，将数据写入到channel，注意只有position和limit之间的数据才会被写入。因此与Scattering Reads相反，Gathering Writes能较好的处理动态消息。

通道之间的数据传输

两个FileChannel可以通过transferTo()和transferFrom()方法相互传输数据
transferFrom方法可以从一个channel传输数据到当前FileChannel中。

Selector

仅用单个线程来处理多个channel的好处是，只需要更少的线程来处理通道。对于操作系统来说，线程之间上下文切换开销很大，而且每个线程都要占用系统一些资源。

通过Selector选择通道

向Selector注册了通道后，可以调用select()方法，方法返回你所感兴趣的事件已经准备就绪的通道。

  • int select() 阻塞到至少一个通道在注册的事件上就绪
  • int select(long timeout) 增加超时时间
  • int selectNow() 不会阻塞，不管什么通道都立刻返回。如果从前一次选择操作后，没有一个通道就绪，则返回0

selectedKeys()

一旦select()方法返回，表明一个或多个通道已经就绪，然后可以通过selector.selectedKeys()方法，访问以就绪的事件。

wakeUp()

线程调用select()方法后进入阻塞状态，此时可以通过另一个线程调用Selector.wakeUp()方法，阻塞的select()方法会立即返回。
如果当前线程没有阻塞在select()方法，那么下个调用select()方法的线程会立即wakeUp。

close()

用完Selector后调用起close()方法会关闭该Selector，但通道本身并不会关闭。

Java IO

By ZhaoLoON

发表于 2015-05-04

最近在并发编程上看到一些关于Java基础知识的文章，抽空看了下，算作是复习吧。

参考：

概述

抽象类InputStream、OutputStream、Reader、Writer有很多子类，其中InputStream和OutputStream是用来处理字节流，Reader和Writer用来处理字符流。

  • InputStreamReader将字节流转换为字符流，读入的字节流可以通过设定的charset转换为字符流。

  • OutputStreamWriter将写入的字符流转换为字节流，写入的字符流可以通过设定的charset转换为字节流。

  • 读取文件可以根据是二进制文件还是文本文件选择不同的类，二进制文件使用FileInputStream，文本文件使用FileReader。同样，写入文件分别对应FileOutputStream和FileWriter。

  • 如果需要跨越式的读写文件可以使用RandomAccessFile

  • ByteArrayInputStream和CharArrayReader可以讲字节或者字符数组转换为输入流。

  • 字节和字符数组可以通过ByteArrayOutputStream和CharArrayWriter写入，写入完成后，可以通过toByteArray()或者toCharArray()以数组的形式返回。

  • InputStream使用read方法读取数据，read返回一个整数，整数为读取到的一个字节的内容(0~255)，当流读到末尾时，返回-1.

  • Reader的read方法一次返回一个字符，返回值的范围在0~65535之间，文件末尾时返回-1，这并不意味着read一次读取两个字节，read会根据文本的编码一次读取一个或多个字节。

  • 可以说JAVA IO的整个设计模式采用了装饰模式

下图转自并发编程网

InputStream

InputStream是Java IO中所有字节输入流的基类，用于读取字节流，一次读取一个字节。

Java7可以使用try-with-resource结构确保InputStream在使用完后关闭。

InputStream还有两个读取数据并将数据存储到缓冲数组的read方法

  • read(byte[] b)
  • read(byte[] b, int offset, int length)

一次读取一个字节数组的方式比一次读取一个字节的方式快的多

  1. read(byte[] b)

    • 从InputStream中读取一定数量的字节，存入到buffer b中，返回值为int类型，表示读取到的字节数量。
    • 如缓冲b的长度为0，则不会读取任何字节，返回值为0，否则read将尝试至少读取一个字节，如果InputStream中没有任何数据可供读取（已经到达文件末尾），则返回-1。
    • 第一个读取的字节存入b[0]，第二个读取到的字节存入b[1]，依次类推，最多读取b.length个字节。
    • 返回值k， 0<k<b.length， b[k]~b[b.length-1]可能为以前读取到的值，注意按照返回值k解析数据。
    • 等同于read(b, 0, b.length)

  2. read(byte[] b, int offset, int length)

    • 从InputStream中读取length个字节的数据，从offset位置开始写入b中，实际读取的字节数据可能小于length，返回值表示实际读取到的字节数。
    • 第一个读取到的字节存入b[offset]，第二个读取的字节存入b[offset+1]，依次类推。

OutputStream

Java IO所有字节输出流的基类

  1. write(byte[] b)

    将字节数组b写入的输出流，等同于write(b, 0, b.length)

  2. write(byte[] b, int offset, int length)

    从字节数组b的b[offset]开始，写入length个字节到输出流

  3. flush()

    将所有写入到OutputStream的数据冲刷到相应的目标媒介，如果目标媒介有底层操作系统提供，flush将写入的数据传递给操作系统，由操作系统写入物理设备中，flush并不能保证操作系统将实际数据写入到物理设备。

FileInputStream 和 FileOutputStream

以字节流的形式读取写入文件

FileOutputStream(File file, boolean append)构造方法中布尔值append表示是否拼接文件，true表示从文件末尾开始写入，false表示覆盖该文件，从文件头开始写入。

RandomAccessFile

RandomAccessFIle可以来回读写文件，替换文件中的某些部分。

  • 创建RandomAccessFile

“r”:以只读模式打开文件，调用write方法是会抛出IOException
“rw”:读写模式打开文件，若文件不存在，则创建一个新的文件
“rws”:打开以便读取和写入，对于 “rw”，还要求对文件的内容或元数据的每个更新都同步写入到底层存储设备。
“rwd”:打开以便读取和写入，对于 “rw”，还要求对文件内容的每个更新都同步写入到底层存储设备。
rws会每次同步写入数据和元数据，如文件修改时间等，rwd不会，所以rwd可以减少I/O数量，效率比rws更高，当然rw效率要高于rwd

  • RandomAccessFile读写

通过seek()方法可以移动文件指针到任何位置。可以通过getFilePointer()获取当前文件指针的位置。
skipBytes(n)尝试跳过n个字节
length()返回文件的长度

PushbackInputStream

PushbackInputStream用友一个缓冲区，允许在缓冲区未满的情况下，将已经读取的数据推回流的前端。

可以通过构造器设置缓冲区的大小

SequenceInputStream

SequenceInputStream将两个或多个InputStream合并为一个，先读取第一个InputStream，结束后从第二个InputStream起始处开始读取。

Serializable, ObjectInputStream, ObjectOutputStream

  • Serializable接口是一个标记接口，接口并未定义任何方法，用户来标记对象是否可以被序列化

  • ObjectOutputStream可以将实现了Serializable接口的Java对象序列化到文件中

  • ObjectInputStream可以从文件中反序列化出Java对象。

  • 另外值得说的是公司的安全编码规范中也有说到对象序列化，对于敏感数据尽量不要序列化到文件中，可以使用transient来修饰敏感数据域，这样在对象序列化的时候就会忽略这个数据域。

  • 若必须要对敏感数据进行序列化，可以重写ObjectOutputStream的write方法和ObjectInputStream的read方法，在write的时候进行加密，read的时候解密。

Reader

Reader是Java IO中Reader系列类的基类，和基于读取字节的InputStream不同的是，Reader读取字符信息的。也就是说Reader是读取文本文档的，而InputStream是读取原始字节的。

当前很多应用程序使用UTF编码存取文档数据。UTF-8是一种不定长编码，英文字符在UTF-8中采用1一个字节表示，中文字符使用3个字节表示。UTF-16采用固定的2个字节编码字符。因此读取文本文档时，一个字节可能并不代表一个字符。如果仅仅每次读取一个字节，然后转换为字符，最终可能得到的是乱码。

Reader类可以解决这个问题，你只需要告诉Reader以什么编码方式进行解码。

Reader的raed()方法返回读取到的下一个字符的int值，当流结束时，read()返回-1， -1为int值而不是字节或字符值。

Writer和其他Reader、Writer的子类

Writer和其他Reader、Writer的子类基本上可以类比InputeStream、OutputStream的子类。

  • FileReader总是以系统的默认编码方式解码文件，并且无法修改FileReader的解码方式，如果需要使用不同的编码进行解码文件，可以使用InputStreamReader代替FileReader
  • LineNumberReader是BufferedReader的子类，支持读取整行和获取当前读取指针所在的行，值得注意的是LineNumberReader的setLineNumber()方法不会更改流中的当前位置，仅仅更改getLineNumber()方法的返回值。
  • StringReader可以将字符串String转换为Reader，以流的形式读取。
