Java IO

By ZhaoLoON

发表于 2015-05-04

最近在并发编程上看到一些关于Java基础知识的文章，抽空看了下，算作是复习吧。

参考：

  1. Java IO教程
  2. Java IO Tutorial

概述

抽象类InputStream、OutputStream、Reader、Writer有很多子类，其中InputStream和OutputStream是用来处理字节流，Reader和Writer用来处理字符流。

  • InputStreamReader将字节流转换为字符流，读入的字节流可以通过设定的charset转换为字符流。

  • OutputStreamWriter将写入的字符流转换为字节流，写入的字符流可以通过设定的charset转换为字节流。

  • 读取文件可以根据是二进制文件还是文本文件选择不同的类，二进制文件使用FileInputStream，文本文件使用FileReader。同样，写入文件分别对应FileOutputStream和FileWriter。

  • 如果需要跨越式的读写文件可以使用RandomAccessFile

  • ByteArrayInputStream和CharArrayReader可以讲字节或者字符数组转换为输入流。

  • 字节和字符数组可以通过ByteArrayOutputStream和CharArrayWriter写入，写入完成后，可以通过toByteArray()或者toCharArray()以数组的形式返回。

  • InputStream使用read方法读取数据，read返回一个整数，整数为读取到的一个字节的内容(0~255)，当流读到末尾时，返回-1.

  • Reader的read方法一次返回一个字符，返回值的范围在0~65535之间，文件末尾时返回-1，这并不意味着read一次读取两个字节，read会根据文本的编码一次读取一个或多个字节。

  • 可以说JAVA IO的整个设计模式采用了装饰模式

InputStream

InputStream是Java IO中所有字节输入流的基类，用于读取字节流，一次读取一个字节。

Java7可以使用try-with-resource结构确保InputStream在使用完后关闭。

InputStream还有两个读取数据并将数据存储到缓冲数组的read方法

  • read(byte[] b)
  • read(byte[] b, int offset, int length)

一次读取一个字节数组的方式比一次读取一个字节的方式快的多

  1. read(byte[] b)

    • 从InputStream中读取一定数量的字节，存入到buffer b中，返回值为int类型，表示读取到的字节数量。
    • 如缓冲b的长度为0，则不会读取任何字节，返回值为0，否则read将尝试至少读取一个字节，如果InputStream中没有任何数据可供读取（已经到达文件末尾），则返回-1。
    • 第一个读取的字节存入b[0]，第二个读取到的字节存入b[1]，依次类推，最多读取b.length个字节。
    • 返回值k， 0<k<b.length， b[k]~b[b.length-1]可能为以前读取到的值，注意按照返回值k解析数据。
    • 等同于read(b, 0, b.length)

  2. read(byte[] b, int offset, int length)

    • 从InputStream中读取length个字节的数据，从offset位置开始写入b中，实际读取的字节数据可能小于length，返回值表示实际读取到的字节数。
    • 第一个读取到的字节存入b[offset]，第二个读取的字节存入b[offset+1]，依次类推。

OutputStream

Java IO所有字节输出流的基类

  1. write(byte[] b)

    将字节数组b写入的输出流，等同于write(b, 0, b.length)

  2. write(byte[] b, int offset, int length)

    从字节数组b的b[offset]开始，写入length个字节到输出流

  3. flush()

    将所有写入到OutputStream的数据冲刷到相应的目标媒介，如果目标媒介有底层操作系统提供，flush将写入的数据传递给操作系统，由操作系统写入物理设备中，flush并不能保证操作系统将实际数据写入到物理设备。

FileInputStream 和 FileOutputStream

以字节流的形式读取写入文件

FileOutputStream(File file, boolean append)构造方法中布尔值append表示是否拼接文件，true表示从文件末尾开始写入，false表示覆盖该文件，从文件头开始写入。

RandomAccessFile

RandomAccessFIle可以来回读写文件，替换文件中的某些部分。

  • 创建RandomAccessFile

“r”:以只读模式打开文件，调用write方法是会抛出IOException
“rw”:读写模式打开文件，若文件不存在，则创建一个新的文件
“rws”:打开以便读取和写入，对于 “rw”，还要求对文件的内容或元数据的每个更新都同步写入到底层存储设备。
“rwd”:打开以便读取和写入，对于 “rw”，还要求对文件内容的每个更新都同步写入到底层存储设备。
rws会每次同步写入数据和元数据，如文件修改时间等，rwd不会，所以rwd可以减少I/O数量，效率比rws更高，当然rw效率要高于rwd

  • RandomAccessFile读写

通过seek()方法可以移动文件指针到任何位置。可以通过getFilePointer()获取当前文件指针的位置。
skipBytes(n)尝试跳过n个字节
length()返回文件的长度

File

  • 实例化File对象
  • file.exists()检测文件是否存在
  • file.length()获取文件长度
  • file.renameTo(“c:\text1.txt”)，移动或者重命名文件
  • file.delete() 删除文件
  • file.deleteOnExit() 在JVM退出时删除文件
  • file.isDirectory() 检测是文件还是目录
  • file.list() 获取目录下的所有文件与子目录的字符串名称
  • file.listFiles() 返回当前目录下的所有文件与子目录的File对象
  • canExecute(), canRead(), canWrite()检测权限
  • createTempFile(String prefix, String suffix)在系统临时文件目录下创建一个临时文件，返回File对象
  • createTempFile(String prefix, String suffix, File directory)在directory目录下创建临时文件，返回File对象
  • isHidden() 文件是否隐藏
  • lastModified()上次修改时间
  • listRoots() 返回可用的文件系统的根目录
  • mkdir()创建目录
  • mkdirs() 创建目录，包含不存在的父目录
  • setReadOnly(), setReadable(boolean), setWritable(boolean), setExcutable(boolean)设置权限

PushbackInputStream

PushbackInputStream用友一个缓冲区，允许在缓冲区未满的情况下，将已经读取的数据推回流的前端。

可以通过构造器设置缓冲区的大小

SequenceInputStream

SequenceInputStream将两个或多个InputStream合并为一个，先读取第一个InputStream，结束后从第二个InputStream起始处开始读取。

Serializable, ObjectInputStream, ObjectOutputStream

  • Serializable接口是一个标记接口，接口并未定义任何方法，用户来标记对象是否可以被序列化

  • ObjectOutputStream可以将实现了Serializable接口的Java对象序列化到文件中

  • ObjectInputStream可以从文件中反序列化出Java对象。

  • 另外值得说的是公司的安全编码规范中也有说到对象序列化，对于敏感数据尽量不要序列化到文件中，可以使用transient来修饰敏感数据域，这样在对象序列化的时候就会忽略这个数据域。

  • 若必须要对敏感数据进行序列化，可以重写ObjectOutputStream的write方法和ObjectInputStream的read方法，在write的时候进行加密，read的时候解密。

Reader

Reader是Java IO中Reader系列类的基类，和基于读取字节的InputStream不同的是，Reader读取字符信息的。也就是说Reader是读取文本文档的，而InputStream是读取原始字节的。

当前很多应用程序使用UTF编码存取文档数据。UTF-8是一种不定长编码，英文字符在UTF-8中采用1一个字节表示，中文字符使用3个字节表示。UTF-16采用固定的2个字节编码字符。因此读取文本文档时，一个字节可能并不代表一个字符。如果仅仅每次读取一个字节，然后转换为字符，最终可能得到的是乱码。

Reader类可以解决这个问题，你只需要告诉Reader以什么编码方式进行解码。

Reader的raed()方法返回读取到的下一个字符的int值，当流结束时，read()返回-1， -1为int值而不是字节或字符值。

Writer和其他Reader、Writer的子类

Writer和其他Reader、Writer的子类基本上可以类比InputeStream、OutputStream的子类。

  • FileReader总是以系统的默认编码方式解码文件，并且无法修改FileReader的解码方式，如果需要使用不同的编码进行解码文件，可以使用InputStreamReader代替FileReader
  • LineNumberReader是BufferedReader的子类，支持读取整行和获取当前读取指针所在的行，值得注意的是LineNumberReader的setLineNumber()方法不会更改流中的当前位置，仅仅更改getLineNumber()方法的返回值。
  • StringReader可以将字符串String转换为Reader，以流的形式读取。
